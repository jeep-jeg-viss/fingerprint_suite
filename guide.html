Input Test Harness

A comprehensive input capture and visualization system

What is This?

The Input Test Harness is a browser-based tool for capturing, visualizing, and analyzing mouse and keyboard input patterns in real-time. It's designed for:

  • Behavioral Analysis - Studying user interaction patterns
  • Bot Detection Research - Identifying automated vs human input
  • UX Testing - Understanding user navigation patterns
  • Biometric Research - Analyzing typing dynamics and mouse kinematics
  • Security Research - Testing input-based authentication systems

Core Components

1. Mouse Path Tracking & Segmentation

The system captures mouse coordinates continuously using mousemove events. Movement is automatically segmented based on a configurable pause threshold (default: 250ms).

Segmentation Algorithm

Continuous movement forms a "segment". When mouse stops for SEGMENT_TIMEOUT ms, the segment is finalized. Each segment gets:

  • Unique ID (auto-incrementing)
  • Color from rotating palette
  • Start/end timestamps
  • Array of (x, y, t) points

Implementation Details

function startNewSegment(x, y) {
    currentSegment = {
        id: currentSegmentIndex,
        color: PALETTE[currentSegmentIndex % PALETTE.length],
        startTime: getRelativeTime(),
        endTime: null,
        points: [{x, y, t: getRelativeTime()}]
    };
}

// Timeout-based segmentation
timeoutId = setTimeout(() => {
    endSegment();  // Close segment after 250ms pause
}, SEGMENT_TIMEOUT);

2. Click Detection & Visualization

All mouse button events are captured (left/middle/right) with precise coordinates and timing. Visual markers are created using SVG elements with color-coding per button type.

Button Color Code
Left Red (#e74c3c) e.button === 0
Middle Orange (#f39c12) e.button === 1
Right Blue (#3498db) e.button === 2

3. Keyboard Capture & Metrics

The system captures both keydown and keyup events to calculate:

  • Characters Per Second (CPS) - Typing speed
  • Words Per Minute (WPM) - Using 5 chars = 1 word
  • Dwell Time - How long each key is held
  • Keycap Visualization - Visual feedback of pressed keys
The typing metrics only count printable characters. Modifier keys (Shift, Ctrl, Alt, etc.) are tracked in keyEvents[] but excluded from speed calculations.

4. Timing System

Uses performance.now() for high-resolution timing (microsecond precision). All timestamps are relative to sessionStartTime for easier analysis: 

function getRelativeTime() {
    return performance.now() - sessionStartTime;
}

Use Cases

1. Behavioral Biometrics

Research in behavioral biometrics uses this data to create unique user profiles based on typing rhythm, mouse movement patterns, and interaction timing. The segment-based mouse tracking captures acceleration, deceleration, and pause patterns characteristic of human vs automated input.

2. Usability Testing

UX researchers can analyze mouse trajectories to identify confusion points, hesitation patterns, and inefficient navigation paths. Segment colors make it easy to spot where users pause or change direction.

3. Security Analysis

Security teams use this to test bot detection systems. Human input typically shows:

  • Variable mouse velocity (not constant speed)
  • Bezier-curve-like trajectories (not straight lines)
  • Natural pauses in typing (200-300ms between words)
  • Variable key dwell times (50-150ms typical)

4. Accessibility Research

Understanding how users with different abilities interact with interfaces. Mouse path data can reveal motor control patterns and help design more accessible input methods.

Technical Implementation Notes

Canvas Rendering

Mouse paths are drawn on a dedicated canvas overlay (z-index: 10) separate from the click markers (SVG, z-index: 20). This separation allows independent toggling of path and click visibility.

Event Throttling

Mouse events are captured at the browser's native rate (typically 60-120Hz depending on mouse polling rate). No artificial throttling is applied to preserve precision. However, this can generate large datasets during long sessions.

Memory Management

All data is held in memory until cleared or page reload. For long capture sessions, consider periodic exports to prevent memory bloat. Typical memory usage:

  • ~100 bytes per mouse point (x, y, t as numbers)
  • ~500 bytes per key event
  • ~1KB per click event (includes target element info)

Browser Compatibility

Uses modern Web APIs:

  • performance.now() - High-res timing (all modern browsers)
  • navigator.clipboard - For screenshot copy (may require HTTPS)
  • Canvas API - Universal browser support
  • SVG - Universal browser support

Analysis Examples

Calculating Mouse Velocity

function calculateVelocity(segment) {
    const velocities = [];
    for (let i = 1; i < segment.points.length; i++) {
        const p1 = segment.points[i-1];
        const p2 = segment.points[i];
        
        const distance = Math.sqrt(
            Math.pow(p2.x - p1.x, 2) + 
            Math.pow(p2.y - p1.y, 2)
        );
        const time = p2.t - p1.t; // ms
        
        velocities.push(distance / time); // pixels per ms
    }
    return velocities;
}

Detecting Typing Bursts

function findTypingBursts(keyEvents, burstThreshold = 100) {
    const bursts = [];
    let currentBurst = [];
    
    for (let i = 0; i < keyEvents.length; i++) {
        if (keyEvents[i].type !== 'keydown') continue;
        
        if (currentBurst.length === 0) {
            currentBurst.push(keyEvents[i]);
        } else {
            const last = currentBurst[currentBurst.length - 1];
            const gap = keyEvents[i].t - last.t;
            
            if (gap < burstThreshold) {
                currentBurst.push(keyEvents[i]);
            } else {
                if (currentBurst.length > 3) {
                    bursts.push([...currentBurst]);
                }
                currentBurst = [keyEvents[i]];
            }
        }
    }
    
    return bursts;
}

Privacy & Ethics

Important: This tool captures detailed behavioral data that can uniquely identify individuals (behavioral biometrics). When using for research:

  • Always obtain informed consent from participants
  • Clearly explain what data is being captured
  • Store data securely and anonymize when possible
  • Consider GDPR/CCPA compliance for personal data
  • Delete data when no longer needed
This tool runs entirely client-side. No data is sent to any server unless you explicitly export and share the JSON file.
